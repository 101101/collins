Centos 6.5 installation guide

This guide will get you started on a fresh Centos 6.5 box.

Dependencies

This will be all the dependencies needed. rpmforge to get latest git. ruby, mysql-server, unzip, wget is self explanatory. daemon is for starting collins as a service.

wget http://pkgs.repoforge.org/rpmforge-release/rpmforge-release-0.5.3-1.el6.rf.x86_64.rpm
rpm -i rpmforge-release-0.5.3-1.el6.rf.x86_64.rpm
yum install -y ruby mysql-server unzip wget
yum install -y git --enablerepo rpmforge-extras

wget http://libslack.org/daemon/download/daemon-0.6.4-1.x86_64.rpm
rpm -i daemon-0.6.4-1.x86_64.rpm

You need Oracle Java JRE and JCE. Since Oracle has a licence agreement they really want you to read it's hard to wget from their servers. Download JRE from here: Java JRE 6 Download JCE from here: Java JCE 6 Get the files onto your box and run the following: 


mkdir /usr/java
cd /usr/java
# replace with your rpm name
rpm -ivh ~/jre.rpm
# replace with your zip name
unzip JCE.zip
cd UnlimitedJCEPolicy
mv -f * /usr/java/latest/lib/security

Preparing mysql

chkconfig mysqld on
service mysqld start
/usr/bin/mysql_secure_installation

Setup Security

Add iptables rules for collins, or turn off iptables if you don't need it on the box. Remember to disable SELinux if you don't want to use that.

Install Collins

Collins should be installed under /usr/local/collins/current

adduser collins
cd ~
wget http://tumblr.github.io/collins/releases/collins-1.2.4.zip
cd /usr/local
unzip ~/collins-1.2.4.zip
cd collins
# create current dir or make a symlink.
mkdir current
mv * current
cd current
./scripts/setup
# Run the commands the setup script tells you.

Now remember to edit your production.conf file to have what you need. collins_conf_dir = "/usr/local/collins/current/conf"

To start Collins on boot:

cp /usr/local/collins/current/scripts/collins.sh /etc/init.d/collins
chmod +x /etc/init.d/collins
chkconfig --add collins
chkconfig collins on
